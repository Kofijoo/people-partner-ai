This skill is used to create and apply a consistent visual theme to an application or interface.

## When to Use

Use this skill when:
- defining a product theme
- choosing color palettes
- selecting typography pairings
- creating light or dark mode systems
- improving visual consistency across the UI

---

## Core Approach

Before choosing a theme, define:

1. **Product context**
   - What is the product?
   - Who is it for?
   - What should it feel like?

2. **Visual tone**
   - Professional
   - analytical
   - bold
   - calm
   - premium
   - technical
   - friendly
   - minimal

3. **Theme goal**
   - Should the UI feel more trustworthy, modern, energetic, or refined?

---

## Theme Design Rules

A theme should include:

- primary color
- secondary color
- accent color
- background and surface colors
- text colors
- border/muted states
- typography pairing or font direction

Always ensure:
- strong contrast
- readability
- visual consistency
- a clear hierarchy between primary, secondary, and accent usage

---

## Application Rules

When applying a theme:
- use consistent tokens or CSS variables
- map theme values cleanly into the design system
- keep components visually coherent
- make sure charts, badges, cards, and states fit the same theme

---

## Theme Types

You may either:

### 1. Generate a custom theme
Create a theme based on the product and context.

### 2. Apply a named preset theme
If preset themes exist, select the closest match and adapt it as needed.

Custom themes are preferred when the product has a specific identity.

---

## Anti-Patterns

Never:
- use random colors without a system
- overuse accent colors
- create low-contrast interfaces
- mix conflicting visual moods
- apply themes inconsistently across components

---

## Final Rule

A good theme should make the product feel:
- intentional
- polished
- consistent
- readable
- memorable
