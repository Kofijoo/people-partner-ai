This skill is used to design and implement high-quality frontend interfaces (websites, dashboards, components, applications) that feel intentional, memorable, and production-ready.

---

## When to Use
Use this skill when building or improving:
- UI components
- Landing pages
- Dashboards
- Web applications
- Any visual frontend interface

---

## Core Approach

Before coding, define a clear design direction:

- **Purpose** → What problem does this solve? Who is it for?
- **Tone** → Choose a bold aesthetic (minimal, brutalist, luxury, playful, editorial, etc.)
- **Constraints** → Framework, performance, accessibility
- **Differentiation** → What makes this visually memorable?

**Rule:** Commit to a strong direction. Avoid neutral or generic design.

---

## Implementation Rules

Always produce:

- Production-ready code (React, HTML/CSS/JS, etc.)
- Functional and usable interfaces
- Cohesive and intentional visual design
- High attention to detail

---

## Design Principles

### 1. Typography
- Use distinctive, high-quality fonts
- Avoid generic fonts (Arial, Inter, Roboto)
- Pair display + body fonts intentionally

### 2. Color & Theme
- Use a strong, cohesive palette
- Prefer dominant colors with sharp accents
- Use CSS variables for consistency

### 3. Motion
- Add meaningful animations and transitions
- Focus on high-impact moments (page load, hover, scroll)
- Prefer CSS animations or motion libraries (React)

### 4. Layout & Composition
- Avoid standard layouts
- Use asymmetry, layering, spacing, and hierarchy
- Balance density and whitespace intentionally

### 5. Visual Depth
- Use textures, gradients, shadows, overlays
- Avoid flat, lifeless backgrounds
- Create atmosphere

---

## Anti-Patterns (STRICT)

Never use:
- Generic SaaS layouts
- Default fonts and styles
- Overused color schemes (e.g. purple gradients on white)
- Cookie-cutter UI patterns
- Repetitive design across outputs

---

## Complexity Matching

- Maximalist design → richer visuals, motion, effects
- Minimalist design → precision, spacing, restraint

Elegance = correct execution of the chosen style

---

## Final Rule

Every design must feel:
- intentional
- distinctive
- production-ready

Avoid anything that looks auto-generated.
